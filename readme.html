Kriteria Pada Pengumpulan Project Akhir

No Kriteria Dasar Terpenuhi? Catatan
1. Masing-masing elemen wajib berisi konten yang peruntukkannya sesuai dengan elemen tersebut ✅ Semua elemen sudah dikaji sesuai peruntukkannya walaupun masih kurang dalam penerapan semantic html (banyak penggunaan div).
2. Masing-masing elemen wajib berisi konten yang peruntukkannya sesuai dengan elemen tersebut ✅
3. Tidak terdapat duplikasi elemen <main> ✅
4. Elemen <main> harus berada diantara <header> dan <footer> ✅
5. Harus terdapat elemen <header> sebelum <main> dan <footer> sesudah <main>
 ✅
6. Jika terdapat elemen <header> atau <footer> di dalam elemen <article> atau <aside>, maka hal tersebut diperbolehkan ✅
7. Wajib menampilkan menampilkan foto di elemen <aside> ✅
8. Menyusun layout menggunakan flexbox, bukan float ✅
9. Tema yang ditampilkan bebas, kecuali tema Bandung
 ✅



No Penilaian Tambahan Terpenuhi? Catatan
1. Menerapkan tampilan aplikasi yang menarik ✅ Menurut kami, aplikasi web ini sudah dibuat semenarik mungkin, seperti warna tema yang sesuai, tidak ada konten bertumpuk, font yang tepat, dan padding-margin yang tepat
2. Menerapkan layout yang responsif ✅ Penerapan layout yang responsif dan menyesuaikan lebar viewport. Adapun tingkat responsif yang optimal dapat dicapai pada viewport minimal 525px dan maksimal 2560px (Jika diluar range ini, mungkin tampilan website agak sedikit berantakan meskipun masih sangat readable)
3. Terdapat penerapan JavaScript dalam memanipulasi DOM ✅ Website ini berbasis JavaScript yang mana terdapat banyak sekali fitur yang bisa anda eksplorasi sendiri



No LARANGAN Terpenuhi? Catatan
1. Menggunakan library Bootstrap, Materialize, Foundation, ataupun library lain yang semacamnya untuk menyusun layout ✅ Website ini sama sekali tidak menggunakan library apapun
2. Menggunakan template project yang didapat selain dari latihan Dicoding ✅ Website ini dibuat tidak menggunakan template apapun, namun mungkin terdapat beberapa komponen (terutama fungsi javascript) yang memiliki kesamaan dengan yang beredar di internet. Hal ini dikarenakan pembuatan website ini dibantu dengan surfing, stackoverflow, reddit, W3school, quora, dan masih banyak yang lainnya
3. Menggunakan tema mengenai Kota Bandung, maupun mengirimkan ulang beberapa project yang dipelajari pada modul latihan ✅ Tidak menggunakan tema terkait, maupun project yang pernah dikumpulkan pada modul latihan



FITUR-FITUR



    --FETCH DATA--
  1. Fetch data dari lokal JSON
  2. Fallback fetch data dari konten github raw data jika gagal mengambil file JSON akibat kebijakan CORS.
  3. Fallback fetch data dari file alternative_data.js dengan cara mengambil variabel global di file alternative_data.js


    4. --DIALOG MODAL--
  4. Custom Confirmation Dialog Modal yang dibuat murni dari javascript tanpa library
  5. Ucapan greetings modal di awal data sebagai sambutan pengunjung saat pertama kali berkunjung ke Website


    6. --DOM & STYLE MANIPULATION--
  6. Pembuatan Header yang dinamis dengan manipulasi DOM sehingga efisien (hanya cukup satu fungsi kode untuk menampilkan header di seluruh halaman, tanpa harus menulis elemen header di setiap halaman web)
  7. Penerapan Footer yang dinamis dengan manipulasi DOM sehingga efisien
  8. Penerapan Loop pada elemen Item Buku Populer dengan memanipulasi DOM melalui javascript
  9. Penerapan Loop pada elemen Layanan dengan memanipulasi DOM melalui javascript
  10. Manipulasi DOM pada pemuatan data about_us
  11. Penerapan ukuran font yang dinamis pada elemen tertentu mengacu pada viewport pengguna
  12. Manipulasi CSS dan class CSS dengan javascript untuk kebutuhan user interact dan responsif
  13. Halaman SIGN IN yang mewajibkan user untuk memasukkan data sesuai dengan ketentuan
  14. Menu Navbar yang menyesuaikan device user (responsif)
  15. Scroll Header yang bersifat dinamis ketika discroll
  16. Animasi SCROLL yang dibuat murni dengan javascript vanilla tanpa library apapun


    17. --LOCALSTORAGE & SESSIONSTORAGE--
  17. Penerapan Local Storage dalam menyimpan data buku user
  18. Penerapan sessionStorage dalam membedakan user interact


    19. --ROUTING HANDLER & JAVASCRIPT BASED WEB--
  19. Setiap Halaman akan dimuat sebagai DOM tanpa harus berpindah window.location Sehingga tidak ada reload dan url hanya tertuju pada satu halaman saja


  20. Dan masih banyak fitur yang lainnya yang bisa anda eksplorasi sendiri


    21. NOTE: Web ini dibuat tanpa menggunakan library apapun (Murni vanilla), hanya berbasis javascript. Semua fitur bisa anda analisis pada file "assets/js/main.js" yang sudah saya beri sedikit komentar untuk memahami struktur fungsi web.
